## 四、备赛方法论（实战可落地）

### 1. 团队分工（"一人主导，两人辅助"）
三条线并行：**硬件制作 / 电控软件 / 文档总结**，视觉算法可由电控或单设一人。你作为电控负责人，主线是"软硬件联调 + K230 通信整合"。原则：禁止"一人打工二人围观"，把项目拆成**电机驱动、屏幕调试、循迹识别、视觉通信**等独立模块并行推进，最后由你整合。

### 2. 72 小时（四天三夜）节奏
- **D1（论证+硬件）**：定 2~3 套总体方案做比较（报告要用），硬件同学最累，当天把杜邦线接出能跑的最小系统。
- **D2（电控攻坚）**：闭环 PID、循迹、姿态全部打通；视觉端 K230 同步出"识别正确率"。
- **D3（联调+固化）**：杜邦线验证 OK 后**立刻焊到洞洞板固定**，避免接触不良。"如果不是接触不良我就过了"是测评现场最高频的悔话。
- **D4（文档+复测）**：文档同学最后冲刺，留足复测与录像时间。
- 铁律：软硬件**同时做**，单元电路尽早固定，**绝不能最后一天才搭系统联调**。

### 3. 常见坑 / 翻车点
- **电机堵转烧驱动**：循迹撞墙 → 堵转电流极大。撞墙立刻抱起断电；测试时**别拖着杜邦线/电池/烧录器**，易卷入轮子。建议软件加堵转/超时保护。
- **接触不良**：杜邦线只用于验证，过关项一律焊死。
- **K230 串口对接**：K230 有 5 个 UART，UART0(小核)、UART3(大核)被占，用户可用 **UART1/UART2/UART4**，需 IOMUX 配引脚；默认波特率 115200。务必约定**带帧头帧尾 + 校验(BCC/和校验)**的协议，防止丢帧错位。
- **10 分钟原则**：单点卡 10 分钟无解立即求助/换路，别盲调耗时。

### 4. 模块化预制（赛前码库）
赛前把这些做成可复用驱动并验证：编码器测速、增量式/位置式 PID、速度环+航向环+转弯环串级、IMU(MPU6050 DMP / 陀螺仪积分)、灰度循迹、OLED 多页调试、蓝牙(JDY-31)在线调参、VOFA+ 上位机波形、K230 收发帧解析。建议直接 fork 上面 `mspm0g3507-car-kit` / `ZLC` 库做底座。

### 5. 器件冗余
清单一出**立即下单**（赛题发布后常断货），主控、电机、驱动、传感器**各多备份**；硬件精度不足会直接卡死指标，宁可超配。

### 6. 论文 / 设计报告要点
- 正文 **A4 8 页以内** + 首页 **300 字以内中文摘要**；正文小四宋体、行距固定值 22 磅。
- **匿名**：封面与每页不得出现学校/姓名/队号，每页上方留 **≥3cm** 空白且空白区无文字，否则取消评审资格。
- 结构：系统设计 → 单元电路设计 → 软件设计 → 系统测试 → 结论；**总体方案给 2~3 种做分析比较**。
- 评分六维：方案设计与论证、理论计算、电路图及设计文件、测试方法与数据、结果分析、报告工整性。报告**与硬件并行写**，别留到最后一天。

### 7. 现场测评 + 心态体力
- 测评前**自测复现多遍 + 录像存证**；进场带齐工具（电烙铁/助焊剂/吸锡、万用表、示波器、备用电池与器件）。
- 排班轮休保证体力，避免通宵导致联调出错；保持"先能跑 60 分，再抠满分"的稳健节奏。

> 你的最短路径：fork `2262727886-stack/mspm0g3507-car-kit` 起底座 → 用 `ZLC` 库与其设计报告对照补外设 → K230 端照 CanMV 官方 UART 例程 + 巡线(线性回归)落地 → 按上面协议定帧 → 三线并行进 72 小时节奏。

## 🔑 关键要点（速记）

- 首选精读两个最对口仓库：ZhijianLi2003/ZLC_MSPM0_Peripheral_Library(2024 H题国一，含设计报告)与 2262727886-stack/mspm0g3507-car-kit(自带K230视觉追踪+通信帧+串级PID+蓝牙调参)，可直接做工程底座。
- K230 串口对接要点：可用 UART1/UART2/UART4(UART0小核、UART3大核被占)，默认波特率115200，务必用带帧头帧尾+校验(BCC/和校验)的协议防丢帧；官方CanMV UART例程与巡线(线性回归)例程是落地基础。
- 团队三线并行：硬件制作/电控软件/文档总结，'一人主导两人辅助'，把电机驱动、循迹、视觉通信拆成独立模块并行，你作为电控负责人主抓联调与K230整合。
- 72小时铁律：软硬件同时做、单元电路尽早固定，杜邦线只用于验证、过关项立即焊死(防接触不良)，绝不能最后一天才搭系统联调。
- 最高频翻车点：电机堵转烧驱动(撞墙立即抱起断电+软件加堵转保护)、接触不良、拖线卷轮、视觉串口丢帧错位；单点卡10分钟无解立即求助换路。
- 器件清单一发布立即下单(常断货)，主控/电机/驱动/传感器各多备份，硬件精度不足会直接卡死指标。
- 设计报告硬规则：正文A4 8页内+首页300字内中文摘要，全程匿名(每页留≥3cm空白、不得出现学校姓名队号否则取消评审)，总体方案给2~3种比较，报告与硬件并行写。
- 视频教程路线：STM32练手跟江协科技(358集)或铁头山羊(HAL库平衡车)；K230视觉跟'速成K230'与官方文档；进阶底层排错看痞子衡。
- STM32练手与MSPM0主力可共用同一套控制思想(增量式PID+速度环/航向环/转弯环串级)，迁移成本低；danshoujieyi/TI-MSPM0G3507提供裸机/FreeRTOS/RT-Thread三套模板。
- 送药小车/双车协作题型参考 CXCXHH 双车协同仓库(MSPM0+STM32+ZigBee)与21年F题'双K210+OpenMV无停留识别'架构(识别与运动解耦)。
