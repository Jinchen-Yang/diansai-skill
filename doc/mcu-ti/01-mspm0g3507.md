# MSPM0G3507 关键参数 + 外设清单

> 参数来源：TI 数据手册《MSPM0G350x 具有 CAN-FD 接口的混合信号微控制器》（已下载到本目录 `MSPM0G3507-datasheet.pdf`）+ TI 产品页。
> 用于 select-parts / interconnect / firmware-scaffold 引脚与外设规划。

## 整体定位

**MSPM0G3507 = Arm Cortex-M0+ @ 80MHz 的 mixed-signal MCU**，主打"数字 + 强模拟 + 运动控制外设"一体，是电赛循迹/运动控制小车的主力主控。全称器件如 `MSPM0G3507SRGZR`（VQFN48）/`...SPMR`（LQFP64）等。

## 核心参数表

| 类别 | 规格 | 备注（电控视角） |
|---|---|---|
| **内核** | Arm **Cortex-M0+** 32-bit，最高 **80 MHz**，带 MPU | **无硬件 FPU**！浮点 PID 偏慢，需 MATHACL/定点优化 |
| **时钟** | 内部高速 SYSOSC（出厂 32MHz，可校准）+ 可选外部晶振；PLL 拉到 80MHz | SysConfig 默认常配 32MHz，**需手动拉满 80MHz** |
| **Flash** | **128 KB**，带 ECC | |
| **SRAM** | **32 KB**，带奇偶校验 | |
| **ADC** | **2× 12-bit、4 Msps、可同步采样**，最多 17 路外部通道；硬件平均可达 ~14bit@250ksps | 双 ADC 同步采样适合电流/相电压同步测量 |
| **DAC** | **1× 12-bit、1 Msps**，带输出缓冲 | 生成波形/基准 |
| **比较器 COMP** | **3× 高速比较器**，高速模式传播延迟 ~32ns，内置参考 DAC（8-bit） | 过流保护、过零检测 |
| **运放 OPA** | **2× 零漂移零交叉斩波运放**（~0.5µV/°C，PGA 最高约 32×）+ **1× 通用运放** | 电流采样信号链可片内完成 |
| **温度传感器** | 内置 | |
| **定时器** | **共 7 个**：2× 16-bit **高级控制定时器（TIMA，带死区/互补）**、1× 32-bit 通用、若干 16-bit 通用（TIMG）；其中**一个通用定时器支持 QEI 正交编码器接口** | TIMA → 电机互补PWM；TIMG-QEI → 编码器测速 |
| **PWM** | **最多 22 路** | 足够多电机 + 舵机 |
| **UART** | **4×**（其中 1 路支持 LIN/IrDA/DALI 等扩展功能） | 留 1 路 printf 调试，1 路收 K230 |
| **SPI** | **2×**，最高 32 Mbit/s | |
| **I2C** | **2×**，支持 FM+ 1 Mbit/s | IMU/OLED |
| **CAN** | **1× CAN 2.0A/B + CAN-FD**（最高 5Mbps） | G 系列独有；多车/底盘通信 |
| **DMA** | **7 通道** | ADC/UART 减负 |
| **数学加速** | **MATHACL**：硬件 DIV / SQRT / MAC / 三角函数（CORDIC 类） | 弥补无 FPU，加速 PID/解算 |
| **安全/加密** | AES、CRC、TRNG、RTC、WWDT | |
| **GPIO** | 最多 **60**（随封装 24/28/44/60）；含 2 路 5V 容忍开漏、若干 20mA 高驱动脚 | |
| **封装** | LQFP64(PM) / LQFP48(PT) / VQFN48(RGZ) / **VQFN32(RHB)** / VSSOP28(DGS) | 小车常见 LQFP48/64 或立创板 |
| **供电** | **1.62 – 3.6 V** 单电源 | 3.3V 系统 |
| **温度** | -40 ~ 125 °C（部分 85°C 档） | |

## 外设地图（小车常用的怎么对应）

| 小车功能 | 用到的外设 | DriverLib 模块前缀 |
|---|---|---|
| 左右轮调速 | TIMA 高级定时器互补 PWM + 死区 | `DL_TimerA_` |
| 编码器测速 | TIMG-QEI 或 通用定时器输入捕获 / GPIO 中断 | `DL_TimerG_` / `DL_GPIO_` |
| 灰度/电池电压采集 | ADC（多通道，可 DMA） | `DL_ADC12_` |
| 收 K230 视觉偏差 | UART（115200/460800，DMA/中断 + 环形缓冲） | `DL_UART_` |
| printf 调试 / VOFA+ | 另一路 UART | `DL_UART_` |
| IMU / OLED | I2C 或 SPI | `DL_I2C_` / `DL_SPI_` |
| 过流保护 | COMP + 内部参考 DAC | `DL_COMP_` |
| 电流采样调理 | OPA（PGA） | `DL_OPA_` |
| 浮点 PID 加速 | MATHACL | `DL_MathACL_` |

## 电控注意点（务必记住）

1. **无 FPU**：`float` PID 走软件浮点慢；用 **MATHACL** 加速除法/平方根/三角，或用 **IQMath** 定点库（SDK 自带，可挂 MATHACL 硬件）。
2. **默认主频 32MHz**：SysConfig 里要手动把 SYSPLL 配到 **80MHz**，否则白白浪费算力。
3. **QEI 通常只有一路**：双电机闭环时，一路用硬件 QEI，另一路用定时器输入捕获 / GPIO 双边沿中断模拟（见 04）。
4. **引脚强复用**：用 IOMUX(PINCMx)，同一物理脚多功能复用，**必须用 SysConfig 冲突检测核对**（详见 04 引脚复用陷阱节）。
5. **3.3V 逻辑**：与 5V 外设/编码器接口注意电平；仅少数脚 5V 容忍。
