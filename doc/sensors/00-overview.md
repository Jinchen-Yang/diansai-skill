# 小车感知方案选型总览（怎么选 / 接什么口 / 用哪个库）

> 资料采集层。需求来源仍以 `design/problem.yaml` 为准；本目录只回答「这类传感器怎么用、接什么口、用哪个库、有什么坑」。
> 深度选型/公式见 `kb/05-感知传感器选型.md`；本篇是「按接口与用途的工程速查」，按需展开到各分册。

## 一、四类感知如何分工（一句话记忆）

电赛小车的反馈 = **横向偏差 + 航向/姿态 + 纵向位移 + 距离**，多源解耦最稳：

| 维度 | 任务 | 主力传感器 | 兜底/补充 |
|---|---|---|---|
| 横向 | 循迹纠偏（黑线/白线/铜线） | 灰度阵列 / 电磁线圈 / K230 视觉中线 | 三选一或加权切换 |
| 朝向 | 走直线、定角度转弯 | IMU 的 yaw（航向角） | 编码器轮速差（漂移大，只做兜底） |
| 纵向 | 走 N 米、定距停车 | 增量编码器里程计 | — |
| 距离 | 避障、到边、对位 | TOF（VL53L0X/L1X）/ 超声 HC-SR04 | 红外（只判有无） |

> 解耦原则：**编码器只管"走了多远"，IMU 只管"朝哪"**，循迹只管"左右偏多少"，测距只管"前方多远"。每个量交给最擅长的传感器，不要互相猜。

## 二、按接口分类（决定占用主控哪种引脚 + 能不能共总线）

| 接口 | 典型传感器 | 占用主控资源 | 能否多设备共用 |
|---|---|---|---|
| **ADC（模拟量）** | 模拟灰度阵列、电磁线圈检波输出 | 每路一个 ADC 通道（MSPM0 17 路够用） | 共用 ADC 模块，多通道扫描+DMA |
| **GPIO（数字量）** | 数字灰度(0/1)、红外避障、按键、有源蜂鸣器 | 每路一个普通 IO | 各占一脚 |
| **I2C** | MPU6050/ICM、VL53L0X/L1X、AS5600、SSD1306 OLED | 仅 2 脚(SDA/SCL)，多设备挂同一总线 | **可，靠地址区分**（地址冲突要 XSHUT/AD0 改址） |
| **SPI** | ICM-42688-P、SSD1306(SPI 版) | SCK/MOSI/MISO + 每设备一个 CS | 可，靠 CS 区分，速率高 |
| **UART** | JY901S(直出角度)、智能灰度(UART 版)、K230 视觉 | 每设备一对 TX/RX | 一对一（或加 RS485 总线） |
| **定时器(QEI/输入捕获)** | 增量编码器(QEI)、超声 Echo 测脉宽(IC)、无源蜂鸣器(PWM) | 一个定时器通道组 | 各占定时器资源 |

> 选型时务必和主控引脚复用能力对上（见 `contracts/mcu/MSPM0G3507.yaml` / `contracts/pinmap.yaml`）：模拟量必须落在带 ADC 的脚；编码器必须落在带定时器 QEI 的脚；I2C/SPI/UART 落在对应外设的复用脚。

## 三、按用途的"省事/主力/练手"三档结论

- **循迹**：练手 = TCRT5000 自拼数字管；主力 = 感为 8 路**模拟**灰度（接 ADC）；省算力 = **I2C/UART 智能灰度**（模块内部直出中线偏差）。电磁题 = 10mH+6.8nF LC 选频 + 运放 + BAT54S 峰值检波 + 差比和，**MSPM0 可用内置 2× OPA + 3× COMP 省外围运放**。
- **IMU**：练手 = MPU6050(I2C)；省事主攻软件 = **JY901S(UART 直出 roll/pitch/yaw)**；高动态自解算 = **ICM-42688-P(SPI 24MHz)**。
- **测距**：精准 = **VL53L0X(2m)/VL53L1X(4m,50Hz)** I2C；省钱粗测 = HC-SR04；只要避障触发 = 红外。
- **测速/里程**：电机自带霍尔正交编码器 → 定时器 **QEI 模式**；旋转角度高精度 = **AS5600(I2C 磁编码)**。
- **人机**：显示 = SSD1306 OLED(I2C 0x3C)；调参监视 = VOFA+ 走串口；输入 = 按键(软件消抖)；提示 = 有源蜂鸣器(GPIO) / 无源(PWM)。
