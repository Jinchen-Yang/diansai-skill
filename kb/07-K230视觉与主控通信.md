## 一、K230 平台与生态速览（选型依据）

### 核心参数（立创·庐山派 / CanMV-K230）

| 项目 | 参数 | 工程含义 |
|---|---|---|
| CPU | 双核玄铁 **C908 RISC-V**，大核 1.6GHz（带 RVV1.0 矢量）+ 小核 0.8GHz | 大核跑 CanMV/MicroPython，小核跑 RTOS/调试 |
| NPU | 第三代 **KPU**，**6TOPS**（INT8/INT16，官方口径 "等效 6T"） | 推理是 K210 的 ~13.7 倍 |
| 内存 | 板载 **512MB LPDDR3**（部分型号 LPDDR4） | 可跑 YOLOv8n/YOLO11n |
| 视频输入 | 3 路 MIPI-CSI，最大 4K；标配 **GC2093** sensor（200 万像素，比 OV5647 画质好） | 多路摄像头可同时输入 |
| 视频输出 | MIPI-DSI，板载 **3.5 寸 LCD**（部分版本 800×480 / 480×800） | 现场免接电脑调阈值 |
| 典型性能（INT8） | MobileNetV2 ≥670fps，ResNet50 ≥85fps，**YOLOv5s ≥38fps** | 实时检测够用 |
| 整板功耗 | < 2W，百毫秒级快启 | 电赛上电即用，不怕断电重启 |
| 开发方式 | **CanMV IDE**（基于 OpenMV）+ MicroPython；兼容 OpenMV 视觉算法库 | 上手快，30+ AI 视觉函数 |

> 注意：CanMV-K230 的 MicroPython API 与 K210 **不完全兼容**（尤其摄像头 sensor、UART 的 FPIOA 引脚配置方式不同），抄 K210 代码会报错。务必照 K230 文档。

### K230 vs OpenMV vs 树莓派/Jetson（电赛取舍）

| 维度 | **K230 (CanMV)** | OpenMV H7 (Plus) | 树莓派 4B | Jetson Nano |
|---|---|---|---|---|
| 主控 | C908 RISC-V 1.6G + 6T KPU | STM32H743 480MHz | A72 4 核 | A57 4 核 + 128CUDA |
| AI 能力 | 强，可跑 YOLOv8n/v5s | 弱（仅 CNN 小模型） | 中（CPU 慢） | 强但功耗高 |
| 启动速度 | **百毫秒级**（电赛友好） | 极快 | 慢（~30s 系统） | 慢 |
| 串口直连单片机 | **原生 UART，最适合** | 原生 UART | 需 USB 转串/GPIO | 同左 |
| 价格 | ~60–130 元 | ~400–700 元 | ~300+ 元 | ~700+ 元 |
| 电赛结论 | **首选**：算力/价格/启动/串口全优 | 任务简单/预算够可选 | 不推荐（启动慢、串口麻烦） | 过重，除非大模型 |

**结论**：循迹追踪类赛题用 K230 做"视觉协处理器"、MSPM0/STM32 做实时电控，是当前性价比最高组合。

## 二、常见视觉任务实现（CanMV / MicroPython）

### 1. 颜色/色块识别（LAB 阈值 + find_blobs）

LAB 阈值格式为 6 元组：`(L_min, L_max, A_min, A_max, B_min, B_max)`，L∈[0,100]，A/B∈[-128,127]。用 CanMV IDE 的 **工具→机器视觉→阈值编辑器**（Threshold Editor）现场取阈值。

```python
import time, os, sys
from media.sensor import Sensor
from media.display import Display
from media.media import MediaManager

sensor = Sensor(width=320, height=240)         # K230 用 media.sensor
sensor.reset()
sensor.set_framesize(width=320, height=240)
sensor.set_pixformat(Sensor.RGB565)
Display.init(Display.ST7701, to_ide=True)      # 板载屏 + IDE 回显
MediaManager.init()
sensor.run()

red_threshold = (30, 100, 15, 127, 15, 127)    # 红色 LAB
while True:
    img = sensor.snapshot()
    blobs = img.find_blobs([red_threshold], area_threshold=150,
                           pixels_threshold=150, merge=True, margin=10)
    if blobs:
        b = max(blobs, key=lambda x: x.pixels())   # 取最大色块
        img.draw_rectangle(b.rect(), color=(255,0,0))
        img.draw_cross(b.cx(), b.cy())
        cx, cy = b.cx(), b.cy()                     # 目标中心坐标→发给单片机
    Display.show_image(img)
```

### 2. 巡线/中线提取（二值化 + 线性回归 get_regression）

```python
GRAYSCALE_THRESHOLD = [(0, 64)]   # 黑线灰度阈值；浅底深线
while True:
    img = sensor.snapshot().to_grayscale()
    img.binary(GRAYSCALE_THRESHOLD)             # 可选先二值化
    line = img.get_regression([(255,255)], robust=True)  # 最小二乘拟合
    if line and line.magnitude() > 8:           # magnitude 越大拟合越可靠
        img.draw_line(line.line(), color=127, thickness=2)
        theta = line.theta()                    # 0/180=竖直对齐；接近90=横置
        rho   = line.rho()
        # 偏差：拟合线在图像中部的 x 与图像中心 x 的差
        err = ((line.x1()+line.x2())//2) - img.width()//2
```

中线角度法（更稳）：把图像按行分段，每行用色块/灰度找黑线中点，多行中点拟合出中线，输出 `error = 中线x − 图像中心x` 给 PID。

### 3. 二维码/AprilTag/数字识别

- **二维码**：`img.find_qrcodes()` → `.payload()` 取内容（需镜头畸变校正 `lens_corr`）。
- **AprilTag**：`img.find_apriltags()`（**无需畸变校正**），支持 TAG16H5/TAG25H9/**TAG36H11**（最常用）等家族，返回 `id()/cx()/cy()/rotation()`，适合做定位标。
- **数字/字符**：简单场景用模板匹配 `find_template`；复杂用 KPU 跑分类小模型（MNIST/自训练）。

### 4. 目标检测 / YOLO 部署（KPU）

流程：`.pt → ONNX → kmodel`（用 **nncase** 转换，**版本必须与板卡固件匹配**，否则加载失败）。零代码可用嘉楠 **AICube** 平台训练并一键转 kmodel。CanMV 提供 `YOLO` 模块 API，支持 **YOLOv5/YOLOv8/YOLO11** 的分类/检测/分割。电赛推荐 **YOLOv8n（nano）**，更大的 v8s 在 K230 上易跑不动/掉帧。

## 三、K230 ↔ MSPM0/STM32 串口通信（本队重点）

### 1. K230 UART 资源与引脚（FPIOA）

K230 共 5 个 UART：UART0 被小核/调试占用，UART3 常被大核占用，**用户用 UART1/UART2/UART4**。引脚需先用 **FPIOA** 复用。庐山派常用对应：

| 串口 | TXD | RXD | 板上接口 |
|---|---|---|---|
| **UART2（推荐）** | GPIO11 | GPIO12 | GH1.25 座子（1=5V,2=RXD,3=TXD,4=GND）|
| UART1 | GPIO3/40 | GPIO4/41 | 40Pin 排针 |
| UART4 | GPIO48/36 | GPIO49/37 | 40Pin 排针 |

**接线三要点**：① **TX↔RX 交叉**（K230_TXD→MCU_RXD，K230_RXD←MCU_TXD）；② **必须共地 GND-GND**；③ K230 与 MSPM0/STM32 均为 **3.3V TTL**，可直连，不要接 5V/RS232 电平。**波特率推荐 115200**（坐标数据量小且实时；若帧率高/数据多可上 230400 或 460800，需两端一致并实测误码）。

### 2. 帧协议设计（变长 + CRC/校验和 + 重同步）

定长简单但不灵活；变长抗扩展、丢一帧不影响后续。推荐结构：

```
| 0xAA | 0x55 | LEN | FUNC | DATA[LEN] | CHK | 0x0D |
  帧头1  帧头2  长度  功能码   数据负载    校验   帧尾
```

- **双帧头 0xAA 0x55**：降低误同步概率（单帧头易和数据碰撞）。
- **LEN**：DATA 字节数（不含头尾/校验），变长关键。
- **FUNC 功能码**：0x01=色块坐标、0x02=巡线偏差、0x03=目标类别、0x04=状态/握手。
- **DATA**：如坐标 x,y（各 int16，小端）、偏差 error（int16）、类别 id（u8）、置信度（u8）。
- **CHK 校验**：入门用**和校验**（LEN+FUNC+DATA 累加取低 8 位）；高可靠用 **CRC-8**。
- **帧尾 0x0D**：辅助定界，配合状态机重同步。

**抗丢包/重同步**：接收用**状态机**，任何一步不匹配（帧头错、校验错、帧尾错）就**回到找帧头状态**，丢掉半截帧而不卡死；连续找头实现自动重同步。**双向握手**：上电后 MCU 发 `0xAA 0x55 ... FUNC=0x04 data='?' ...`，K230 回 ACK，确认链路与波特率一致再进入工作。

### 3. K230 端（Python）打包发送

```python
from machine import UART, FPIOA
import struct

fpioa = FPIOA()
fpioa.set_function(11, FPIOA.UART2_TXD)
fpioa.set_function(12, FPIOA.UART2_RXD)
uart = UART(UART.UART2, baudrate=115200, bits=UART.EIGHTBITS,
            parity=UART.PARITY_NONE, stop=UART.STOPBITS_ONE)

HEAD = b'\xAA\x55'
TAIL = b'\x0D'

def send_frame(func, payload: bytes):
    length = len(payload)
    body = bytes([length, func]) + payload
    chk = 0
    for b in body:
        chk = (chk + b) & 0xFF          # 和校验
    uart.write(HEAD + body + bytes([chk]) + TAIL)

def send_blob(cx, cy):                  # 功能码0x01：色块坐标
    # int16 小端：x,y
    send_frame(0x01, struct.pack('<hh', cx, cy))

def send_line_err(error):               # 功能码0x02：巡线偏差
    send_frame(0x02, struct.pack('<h', error))

def send_target(class_id, conf):        # 功能码0x03：目标类别+置信度
    send_frame(0x03, bytes([class_id & 0xFF, conf & 0xFF]))

# 主循环里：
# blobs/line 处理后 → send_blob(cx,cy) / send_line_err(err)
```

接收 MCU 指令（K230 也可被动接收，做双向）：

```python
buf = b''
while True:
    d = uart.read()          # 非阻塞，无数据返回 None
    if d:
        buf += d
        # 这里同样用状态机/查找 HEAD...TAIL 解析 ACK 或控制命令
```

### 4. MSPM0G3507 端（C，DriverLib + 中断 + 环形缓冲 + 状态机）

MSPM0G3507 的 UART（如 UART0）特性：独立 **4 级深度 TX/RX FIFO**、支持 DMA、过采样 16/8/3、波特率由 **IBRD（整数分频）+ FBRD（小数分频）** 寄存器设置。SysConfig 生成的 `DL_UART_Main_init` 已配好；只需开 RX 中断。

```c
/* ---------- 环形缓冲 ---------- */
#define RB_SIZE 256
static volatile uint8_t  rb[RB_SIZE];
static volatile uint16_t rb_head = 0, rb_tail = 0;

static inline void rb_push(uint8_t c){
    uint16_t n = (rb_head + 1) % RB_SIZE;
    if(n != rb_tail){ rb[rb_head] = c; rb_head = n; }  // 满则丢弃
}
static inline int rb_pop(uint8_t *c){
    if(rb_head == rb_tail) return 0;                   // 空
    *c = rb[rb_tail]; rb_tail = (rb_tail + 1) % RB_SIZE;
    return 1;
}

/* ---------- 串口接收中断（DriverLib） ---------- */
void UART_0_INST_IRQHandler(void){
    switch(DL_UART_Main_getPendingInterrupt(UART_0_INST)){
        case DL_UART_MAIN_IIDX_RX:                       // 收到一字节
            rb_push(DL_UART_Main_receiveData(UART_0_INST));
            break;
        default: break;
    }
}

/* ---------- 解析状态机（变长 + 和校验 + 重同步） ---------- */
typedef enum { S_H1, S_H2, S_LEN, S_BODY, S_CHK, S_TAIL } pstate_t;
#define MAX_DATA 32
typedef struct {
    uint8_t func;
    uint8_t len;
    uint8_t data[MAX_DATA];
    uint8_t ready;            // 1=收到完整有效帧
} frame_t;
static frame_t g_frame;

void protocol_poll(void){
    static pstate_t st = S_H1;
    static uint8_t len=0, func=0, idx=0, sum=0, buf[MAX_DATA];
    uint8_t c;
    while(rb_pop(&c)){
        switch(st){
        case S_H1: st = (c==0xAA)? S_H2 : S_H1; break;
        case S_H2:
            if(c==0x55) st = S_LEN; else st = (c==0xAA)? S_H2 : S_H1;
            break;
        case S_LEN:
            len = c; sum = c; idx = 0;
            st = (len <= MAX_DATA)? S_BODY : S_H1;       // 长度非法→重同步
            if(len==0) st = S_CHK;                        // 兼容功能码无数据
            break;
        case S_BODY:
            if(idx==0){ func=c; sum=(sum+c)&0xFF; buf[idx++]=c; if(len==1) st=S_CHK; }
            else { buf[idx++]=c; sum=(sum+c)&0xFF; if(idx>=len) st=S_CHK; }
            break;
        case S_CHK:
            if(c==sum) st = S_TAIL; else st = S_H1;        // 校验错→丢帧重同步
            break;
        case S_TAIL:
            if(c==0x0D){                                   // 完整有效帧
                g_frame.func = buf[0]; g_frame.len = len;
                for(uint8_t i=0;i<len;i++) g_frame.data[i]=buf[i];
                g_frame.ready = 1;
            }
            st = S_H1;
            break;
        }
    }
}

/* ---------- 主循环用法 ---------- */
int main(void){
    SYSCFG_DL_init();
    NVIC_EnableIRQ(UART_0_INST_INT_IRQN);
    while(1){
        protocol_poll();
        if(g_frame.ready){
            g_frame.ready = 0;
            if(g_frame.func == 0x01){                      // 色块坐标
                int16_t cx = (int16_t)(g_frame.data[1] | (g_frame.data[2]<<8));
                int16_t cy = (int16_t)(g_frame.data[3] | (g_frame.data[4]<<8));
                // → 喂给舵机/电机 PID
            } else if(g_frame.func == 0x02){               // 巡线偏差
                int16_t err = (int16_t)(g_frame.data[1] | (g_frame.data[2]<<8));
                // → PID: turn = Kp*err + Kd*(err-last)
            }
        }
    }
}
```

> 说明：`func` 含在 DATA 里（data[0]=func），坐标从 data[1] 起。STM32 标准库版把 `DL_UART_Main_receiveData/getPendingInterrupt` 换成 `USART_ReceiveData/USART_GetITStatus`，`UART_0_INST_IRQHandler` 换成 `USART1_IRQHandler`，环形缓冲与状态机逻辑完全通用。

### 5. 波特率分频公式（MSPM0）

设 UART 时钟 $f_{clk}$、过采样 $OSR$（16/8/3）、目标波特率 $B$：

$$Divisor = \frac{f_{clk}}{OSR \times B},\quad IBRD=\lfloor Divisor \rfloor,\quad FBRD = \mathrm{round}((Divisor - IBRD)\times 64)$$

例：$f_{clk}=40\text{MHz},OSR=16,B=115200 \Rightarrow Divisor=21.70$，IBRD=21，FBRD=round(0.70×64)=45。一般用 SysConfig 自动算好，理解原理便于排查"乱码=波特率不匹配"。

## 四、工程落地建议（联调避坑）

1. **先回环再互联**：K230 自发自收（TX 接自己 RX）验证 UART 正常；MCU 同理；再交叉连。
2. **乱码排查顺序**：波特率不一致 → 没共地 → TX/RX 没交叉 → 电平不对 → 数据位/停止位不符。
3. **不要在中断里做解析**：中断只 `rb_push`，主循环 `protocol_poll`，避免丢字节。
4. **变长帧 LEN 一定要做合法性检查**（>MAX_DATA 直接重同步），否则野指针/越界。
5. **坐标用 int16 小端**统一约定，置信度/状态用 u8，省带宽、跨平台无歧义。
6. **K230 发送频率**别盲目拉满，30–50Hz 对小车足够，过高反而挤占 MCU 解析。

## 🔑 关键要点（速记）

- 选型：循迹追踪类赛题用 K230(6TOPS/RISC-V/百毫秒快启/原生UART/约60-130元) 做视觉协处理器 + MSPM0/STM32 做实时电控，是性价比与启动速度最优组合，胜过 OpenMV(贵且AI弱)与树莓派/Jetson(启动慢、串口麻烦)。
- K230 串口：用户用 UART1/UART2/UART4（UART0调试、UART3大核占用）；庐山派 UART2=GPIO11(TXD)/GPIO12(RXD)，须先用 FPIOA.set_function 复用引脚，再 machine.UART 初始化。
- 接线三铁律：TX↔RX 交叉、必须共地 GND、双方都是 3.3V TTL 可直连；乱码排查顺序=波特率→共地→交叉→电平→数据/停止位。波特率推荐 115200，数据大可上 230400/460800（两端一致）。
- 帧协议推荐变长结构：双帧头 0xAA 0x55 + LEN + FUNC功能码 + DATA + 和校验/CRC8 + 帧尾 0x0D；双帧头降误同步，LEN 必做合法性检查，功能码区分色块坐标/巡线偏差/目标类别/握手。
- 抗丢包靠状态机重同步：任一步（帧头/校验/帧尾）不匹配立即回到找帧头状态，丢半截帧而不卡死；上电先做 ACK 双向握手确认链路与波特率。
- MCU 接收架构：UART RX 中断里只把字节 push 进环形缓冲(ring buffer)，主循环里 protocol_poll 跑解析状态机——切忌在中断里解析，否则丢字节。
- 数据约定：坐标 x/y 用 int16 小端，置信度/类别/状态用 u8，省带宽且跨平台无歧义；发送频率 30-50Hz 对小车足够，过高反挤占 MCU。
- MSPM0G3507 UART 有 4 级 FIFO、支持 DMA，波特率由 IBRD(整数)+FBRD(小数,×64) 分频；用 DL_UART_Main_getPendingInterrupt/receiveData + DL_UART_MAIN_IIDX_RX 处理中断，SysConfig 自动算分频。
- 视觉算法：颜色识别 find_blobs 用 6 元组 LAB 阈值(IDE阈值编辑器取值)；巡线用 get_regression 取 theta/rho/magnitude，偏差=中线x−图像中心x 喂 PID；AprilTag(TAG36H11,无需畸变校正)做定位、find_qrcodes 取 payload。
- YOLO 部署：pt→onnx→kmodel 用 nncase 转换，nncase 版本必须与板卡固件严格匹配；推荐 YOLOv8n(nano)，可用嘉楠 AICube 零代码训练并一键转 kmodel。
