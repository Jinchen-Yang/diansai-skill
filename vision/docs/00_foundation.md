# 第 0 模块：相机、显示与 UART 基础

## 写代码前先学

请先在 CanMV IDE for K230 v4.0.7 逐个运行亚博原厂示例：

1. `02.Basic/01.fpioa.py`：理解 K230 的外设功能映射，不需要背引脚号。
2. `02.Basic/05.uart.py`：观察 `YbUart(baudrate=115200)`、`send()` 与 `read()`。
3. `02.Basic/17.sensor.py`：理解 `Sensor -> Display -> MediaManager` 的启动和释放顺序。
4. 绘制图像中的“矩形、十字准心、文字”：后续每个视觉模块都靠它们检查算法输出。

原始资料目录：`D:\田浩楠\电赛\K230\K230视觉模块\程序源码`。

## 本模块逻辑

```text
初始化 UART
初始化 Sensor、Display、MediaManager
循环：取图 -> 画中心十字/FPS -> 显示 -> 每 30 帧发一次握手帧，并每 3 帧发一次在线 AIM 帧
停止：释放 UART -> 停相机 -> 关显示 -> 释放媒体缓冲
```

`main.py` 不识别任何目标。它的意义是先把所有后续模块共同依赖的硬件路径验证干净。

## 运行方式

1. 第 0 模块只需在 CanMV IDE 打开并运行 `main.py`；无需上传 `core/` 或 `protocol.py`。
2. 完整库模块才需要把 `core/`、`protocol.py` 与任务脚本一起部署到 K230。
3. 预期现象：板载屏幕和 IDE 均有实时画面；画面中央有红色十字；左上角显示 FPS。
4. 若已接主控或串口助手，每约一秒可收到 `handshake(0x23)`；每约 3 帧收到
   `aim_offset(dx=0, dy=0, found=0)`。后者表示 K230 在线但未运行目标识别。

## UART 接线与验证

使用 3.3V TTL，K230 TX 接主控 RX，K230 RX 接主控 TX，GND 必须共地。先只验证 K230 发出数据，主控端不需要回应。

若图像正常但串口没有帧，先单独运行原厂 `02.Basic/05.uart.py`；若原厂例程也无数据，再检查 K230 所用串口接口、线序和共地，而不是先改协议代码。现有练习车的 MSPM0 固件只把 `aim_offset` 帧计为 K230 在线，因此 PC 端应在 VOFA 的 `I3(k230_age_ms)` 观察该数值回落，而不是期待 UART0 直接打印 K230 的原始帧。

## 通过标准

- 可持续运行 5 分钟，不报 `MediaManager` 或内存错误。
- IDE 与板载屏幕都有图像，FPS 不持续降为 0。
- 串口助手或主控可稳定看到周期性握手帧。

通过后再进入第 1 模块：色块识别。届时需先学习“颜色识别”“多颜色识别”“CV_lite 色块识别（灰度/彩色）”“曝光调节”。

## 现场标定参数（比赛时必看）

本模块不存在算法阈值，不要在现场为追求帧率而随意改分辨率、屏幕初始化顺序或 UART 波特率。只在相机无画面、预览比例错误或 UART 连接异常时，回到亚博对应基础示例逐项验证；不属于比赛现场的阈值调参。
