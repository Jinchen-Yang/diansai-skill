# 第 1 模块：色块识别

## 写代码前先学

在 CanMV IDE for K230 v4.0.7 先运行亚博源码中的：

1. `05.Color/01.color_recognition.py`：单颜色的 LAB 阈值和 `find_blobs`。
2. `05.Color/03.multi_color_recognition.py`：依次检测多种颜色与画框标注。
3. `06.cv_lite/2.rgb888_find_blobs.py`：了解 RGB888/cv_lite 的替代路线；本模块暂不使用它。
4. `06.cv_lite/15.rgb888_adjust_exposure.py`：观察亮暗变化会怎样改变色块识别，暂不直接加入检测链路。

## 本模块做什么

`demos/01_color_blob/main.py` 每帧执行：

```text
取 RGB565 图像
  -> 分别按红、绿、蓝 LAB 阈值找候选 blob
  -> 每种颜色只选像素数最大的一个
  -> 过滤过小目标
  -> 连续 3 帧才标记为 STABLE
  -> 叠加框、中心、坐标、FPS 到板载屏幕和 IDE
```

选择“每色最大 blob”是为了让背景碎片或小反光不会抢走主目标。连续帧确认是为后续主控执行动作打基础：看到一帧不等于可以行动。

## 运行与调参

1. CanMV IDE 直接打开并运行 `demos/01_color_blob/main.py`，无需上传其他文件。
2. 拿红、绿、蓝色纸或物体依次进入镜头，观察对应框和中心十字是否稳定。
3. 修改文件顶部 `COLOR_SPECS` 的 LAB 值。原始值只是亚博教程中的起点，不是通用真值。
4. 小噪点被框住时，先提高 `MIN_AREA` 或 `MIN_PIXELS`；目标经常丢失时，再重新调 LAB 阈值和光照，不要先把面积门限降到很小。

## 通过标准

- 单个目标出现时，框中心随目标移动，颜色标签正确。
- 空场或非目标颜色下不频繁出现假框。
- 目标停住后，在 3 帧内显示 `STABLE`。
- 改变距离与左右位置后，`(cx, cy)` 与画面实际位置一致。

本阶段不向主控发送数据。通过后再把稳定色块结果接到共享 `protocol.py`，并进入第 2 模块：几何图形检测。
