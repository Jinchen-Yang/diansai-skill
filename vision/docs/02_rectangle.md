# 第 2 模块：矩形检测

## 写代码前先学

在 CanMV IDE for K230 v4.0.7 先运行亚博源码中的：

1. 绘制图像中的“绘制矩形、绘制十字准心”。
2. `04.Detecting/02.find_rects.py`：`find_rects(threshold=...)` 和四角点输出。
3. `04.Detecting/04.edges.py`：观察边缘检测对光照、阴影和纹理的敏感性。
4. `06.cv_lite/9.grayscale_find_rectangle.py`：理解 cv_lite 灰度方案的参数项；本模块暂不使用它。

## 本模块逻辑

```text
取 RGB565 图像
  -> find_rects(边缘响应阈值)
  -> 丢弃外接框过小的候选
  -> 选择外接框最大的矩形
  -> 画边框、四角点和中心
  -> 连续 3 帧后标记 STABLE
```

矩形检测找的是“边缘围成的几何形状”，不是“某种颜色”。因此白纸、黑框、路标边界等都可能识别；同样，桌面纹理、窗框和屏幕边缘也可能成为假目标。

## 运行与调参

1. CanMV IDE 直接打开并运行 `demos/02_rectangle/main.py`。
2. 先使用一张边缘清晰的白色硬纸板，外围贴深色胶带，背景尽量纯净。
3. 假框太多：提高 `RECT_THRESHOLD`，再提高 `MIN_BBOX_AREA`。
4. 真实矩形漏检：先让纸板更平整、边缘对比更强，再适度降低 `RECT_THRESHOLD`。
5. 倾斜、远近变化、部分遮挡都应分别测试；不能只在正对镜头的理想位置验证一次。

## 通过标准

- 主矩形出现时，边框与四角基本贴合真实边界。
- 纸板倾斜后，四角点随透视变化而移动。
- 空场或移走目标后，`STABLE` 不应继续保持。
- 改变目标距离后，中心坐标变化方向正确。

通过后再进入第 3 模块：圆形检测。圆形与矩形检测的目标、参数和常见误检不同，不建议一开始混写到一个程序里。

## 现场标定参数（比赛时必看）

| 参数 | 含义 | 什么时候调 |
| --- | --- | --- |
| `RECT_THRESHOLD` | `find_rects()` 的边缘响应阈值 | 假矩形多时上调；真矩形边缘清晰但漏检时下调。 |
| `MIN_BBOX_AREA` | 允许作为主目标的最小外接框面积 | 窗框、纹理等小假框太多时上调；近似真目标太小时下调。 |
| `STABLE_FRAMES` | 主矩形连续几帧一致才算稳定 | 后级会依此触发动作时调整。 |

调试顺序：先固定目标与摄像头距离，清理背景强边缘 -> 调 `RECT_THRESHOLD` -> 调 `MIN_BBOX_AREA` -> 最后调 `STABLE_FRAMES`。如果全帧仍不稳定，优先增加目标 ROI，不要无限改阈值。

## CV Lite 备选方案（已实测表现良好）

当前 `demos/02_rectangle/main.py` 可正常运行，但矩形赛题的最终依据尚未定版。经实机试运行，亚博的 CV Lite 灰度与 RGB888 矩形方案均表现不错，作为本模块的备选实现记录。

| 源码 | 使用时机 |
| --- | --- |
| `06.cv_lite/5.grayscale_find_rectangle_with_corners.py` | 需要四个角点，且灰度对比足够时。 |
| `06.cv_lite/6.rgb888_find_rectangle_with_corners.py` | 需要四个角点，且 RGB888 在该场地中更稳定时。 |
| `06.cv_lite/9.grayscale_find_rectangle.py` | 只需矩形定位、不需角点输出时。 |
| `06.cv_lite/10.rgb888_find_rectangels.py` | 只需矩形定位，且 RGB888 实测更稳定时。 |

定版选择顺序：先以对准、倾斜、远近、移走目标四种状态，用同一块矩形面板对比当前方案与 CV Lite 备选方案。以漏检、假框、帧率和角点稳定性为准，而不以单次演示效果定版。
