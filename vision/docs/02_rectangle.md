# 第 2 模块：矩形检测

## 写代码前先学

在 CanMV IDE for K230 v4.0.7 先运行亚博源码中的：

1. 绘制图像中的“绘制矩形、绘制十字准心”。
2. `04.Detecting/02.find_rects.py`：`find_rects(threshold=...)` 和四角点输出。
3. `04.Detecting/04.edges.py`：观察边缘检测对光照、阴影和纹理的敏感性。
4. `06.cv_lite/9.grayscale_find_rectangle.py`：理解 cv_lite 灰度方案的参数项；本模块暂不使用它。

## 本模块逻辑

```text
取 RGB565 图像
  -> find_rects(边缘响应阈值)
  -> 丢弃外接框过小的候选
  -> 选择外接框最大的矩形
  -> 画边框、四角点和中心
  -> 连续 3 帧后标记 STABLE
```

矩形检测找的是“边缘围成的几何形状”，不是“某种颜色”。因此白纸、黑框、路标边界等都可能识别；同样，桌面纹理、窗框和屏幕边缘也可能成为假目标。

## 运行与调参

1. CanMV IDE 直接打开并运行 `demos/02_rectangle/main.py`。
2. 先使用一张边缘清晰的白色硬纸板，外围贴深色胶带，背景尽量纯净。
3. 假框太多：提高 `RECT_THRESHOLD`，再提高 `MIN_BBOX_AREA`。
4. 真实矩形漏检：先让纸板更平整、边缘对比更强，再适度降低 `RECT_THRESHOLD`。
5. 倾斜、远近变化、部分遮挡都应分别测试；不能只在正对镜头的理想位置验证一次。

## 通过标准

- 主矩形出现时，边框与四角基本贴合真实边界。
- 纸板倾斜后，四角点随透视变化而移动。
- 空场或移走目标后，`STABLE` 不应继续保持。
- 改变目标距离后，中心坐标变化方向正确。

通过后再进入第 3 模块：圆形检测。圆形与矩形检测的目标、参数和常见误检不同，不建议一开始混写到一个程序里。
