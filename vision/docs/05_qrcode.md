# 第 5 模块：二维码识别

## 写代码前先学

先在 CanMV IDE 中运行亚博源码：

1. `06.Codes/02.find_qrcodes.py`：理解 `image.find_qrcodes()`、`rect()` 与 `payload()`。
2. `14.export/CanmvIDE-K230/03.find_qrcodes.py`：理解亚博将二维码结果交给下位机的完整方向；本阶段不要接 UART。
3. 预先准备一个内容简单的二维码，可用手机生成文本 `DIANSAI-01` 后直接对着板载摄像头测试。

## 本模块做什么

`demos/05_qrcode/main.py` 使用 RGB565 的 640 x 480 图像调用 `image.find_qrcodes()`。

每个检测结果会得到：

- `payload`：二维码解码出的原始文本。
- `rect`：二维码外接矩形 `(x, y, w, h)`。
- `center_x`、`center_y`：由矩形计算的中心坐标。

程序会标出同一画面里的全部二维码。它只在二维码内容改变或每 30 帧时打印一次，不会每帧向 IDE 控制台刷文本。

## 运行与调试

1. 在 CanMV IDE 直接打开并运行 `demos/05_qrcode/main.py`。
2. 先使用黑白清晰、无镜面反光的二维码，保证二维码完整进入画面。
3. 若识别不到，先拉近二维码、保持镜头正对、提高环境亮度；不要先改代码。
4. 若画面有多个二维码，确认每个绿色框与显示的内容对应正确。

## 通过标准

- 静止二维码能稳定画出绿色外框和红色中心点。
- IDE 控制台输出的 `payload` 与二维码实际内容一致。
- 同时放入两个不同二维码时，能分别显示而非只保留第一个。
- 取走全部二维码后，画面左上角显示 `QR=0`。

## 现场标定参数（比赛时必看）

当前 `find_qrcodes()` 没有开放给本示例的检测阈值，因此不要在赛场乱改算法参数。正确顺序是：先确保二维码完整入镜、镜头尽量正对、光照均匀 -> 再调摄像头与码的距离 -> 最后才在保持跑车需求的前提下考虑降低分辨率以提高帧率。`MAX_PAYLOAD_CHARS` 仅影响屏幕上的文字截断，不影响解码结果；`PRINT_PERIOD_FRAMES` 仅影响 IDE 调试输出频率。
