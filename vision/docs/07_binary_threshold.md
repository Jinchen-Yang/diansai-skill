# 第 7 模块：RGB888 二值化预处理

## 写代码前先学

先在 CanMV IDE 中运行亚博源码：

1. `06.cv_lite/14.rgb888_threshold_binary.py`：本模块的默认实现。
2. `06.cv_lite/13.grayscale_threshold_binary.py`：对比灰度输入方案。

## 本模块做什么

```text
RGB888 图像
  -> 按亮度阈值变为黑白图
  -> 显示二值化结果
```

二值化只是预处理：它不负责找矩形、不负责识别数字、不负责判断目标类别。它的作用是为后续的字符笔画、边缘、模板匹配或模型输入提供干净的黑白前景。

若赛题出现带矩形外框的数字牌，应在集成层中将本模块与第 2 模块的矩形检测组合；两者不再作为默认的同一个模块。

## 现场标定参数（比赛时必看）

| 参数 | 含义 | 什么时候调 |
| --- | --- | --- |
| `BINARY_THRESHOLD` | 像素分为黑/白的亮度分界值 | 前景和背景分不开时调整；光照变化后必须重新确认。 |
| `BINARY_MAX_VALUE` | 二值图中白色像素的值 | 通常保持 `255`，不作为现场调参。 |
| `PRINT_PERIOD_FRAMES` | IDE 控制台打印周期 | 只影响调试输出，不影响二值化结果。 |

调试顺序：先固定相机和现场光照 -> 缓慢调整 `BINARY_THRESHOLD` -> 观察目标是否成为完整的黑/白轮廓 -> 记录可用值。不要先把二值化与矩形、数字识别、PID 混在一起调。
