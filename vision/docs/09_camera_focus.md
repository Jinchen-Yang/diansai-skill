# 第 9 模块：相机对焦能力与清晰预览

## 写代码前先学

本模块首先参考亚博随附的 `02.Basic/17.sensor.py`，确认 `Sensor -> Display -> MediaManager` 的初始化顺序。

外部文档提及 `sensor.focus_caps()` 和 `sensor.auto_focus()`，但亚博随附源码中没有这两个接口的实际例程。当前板端日志已识别摄像头为 `gc2093_csi2`，因此不能假定它一定有自动对焦马达或固件支持，必须先实机探测。

## 本模块做什么

`demos/09_camera_focus/main.py`：

- 调用 `getattr()` 安全检查 `focus_caps` 与 `auto_focus` 是否存在。
- 仅当 `auto_focus` 可调用时，才按 `ENABLE_AUTO_FOCUS` 请求开启自动对焦。
- 显示未处理的 RGB565 实时预览、中心取景框和 AF 状态，不使用算法、变焦或 UART。
- API 不存在、调用失败或镜头硬件不支持时，程序不会因为该接口报错，而是显示 `AF unsupported: manual lens focus`。

## 使用方法

1. 在 CanMV IDE 打开并运行 `demos/09_camera_focus/main.py`。
2. 将印有细小黑字或棋盘格的平面目标放在黄色取景框中央，距离取电赛预计工作距离，例如 50 cm 到 100 cm。
3. 观察 IDE 控制台的两行输出：`focus caps=...` 与 `auto focus before=... after=...`。
4. 若显示 `AF=True` 或其他启用状态，等待画面稳定 2 至 3 秒，再比较近处和远处画面的清晰度。
5. 若显示 `AF unsupported`，只在镜头存在可转动螺纹调焦环时进行手动调焦：小幅、缓慢旋转镜头外圈，寻找文字边缘最锐利的位置。镜头存在胶水固定、锁紧螺丝或完全不可动时，不要强行旋转。

## 现场标定参数（比赛时必看）

| 参数 | 含义 | 什么时候调 |
| --- | --- | --- |
| `FRAME_WIDTH` / `FRAME_HEIGHT` | 对焦预览的图像尺寸 | 只在预览变形或帧率过低时调整；保持宽高比例一致。 |
| `ENABLE_AUTO_FOCUS` | 接口可用时是否请求自动对焦 | 先保持 `True` 进行能力验证；若 AF 来回抖动，改为 `False` 后按镜头实际焦点使用。 |
| 物理镜头焦点 | 镜头与目标距离对应的实际成像焦点 | 在相机安装位置确定后、赛场实物距离下调整；这不是代码参数。 |

调试顺序：先确认画面本身未被拉伸 -> 查看 AF 能力输出 -> 若 AF 可用则等待它稳定 -> 若 AF 不可用才考虑可转动镜头的物理调焦 -> 最后以实际任务距离检查近、中、远三处清晰度。不要在自动对焦开启时同时手动旋转镜头。
