# 第 11 模块：统一视觉结果规范

## 写代码前先学

本模块不引入新的亚博视觉例程。先回看已通过的单项模块，确认它们各自能输出的目标中心、类别、质量或置信度：

- 色块：类别、`cx`、`cy`、面积。
- 圆形：`cx`、`cy`、半径、检测质量。
- AprilTag：家族、ID、`cx`、`cy`、旋转角。
- 巡线：横向误差、路线角度、丢线状态。
- 在线训练模型：类别、检测框、置信度。

## 为什么现在做这个

单项 Demo 的输出形式不同，不能直接交给主控或塞进一个总 `main.py`。整机程序只应接入已实测通过的模块，并将它们先转换为同一种结果格式，再决定显示、UART 发送和控制动作。

本阶段只定义内部结果，不修改 `contracts/protocol.yaml`。UART 帧字段必须等软件组确认收发需求后再冻结，不能由视觉组单方面扩展。

## 内部结果格式

每帧由当前模式产出一个 `VisionResult`。无目标时也必须明确输出 `valid=0`，不能继续沿用上一帧坐标。

```python
{
    "mode": "watch_detect",
    "valid": 1,
    "target_type": "watch",
    "cx": 318,
    "cy": 236,
    "dx": -2,
    "dy": -4,
    "confidence": 86,
    "area": 12400,
    "angle_deg": None,
}
```

字段约定：

| 字段 | 含义 | 无目标时的值 |
| --- | --- | --- |
| `mode` | 当前运行的视觉模式名称 | 保留当前模式名 |
| `valid` | 当前帧是否有可用目标 | `0` |
| `target_type` | 类别、颜色、Tag ID 或其他目标类型 | `"none"` |
| `cx` / `cy` | 统一坐标系中的目标中心 | `0` |
| `dx` / `dy` | 相对画面中心的有符号偏差 | `0` |
| `confidence` | 0 至 100 的置信度或质量评分 | `0` |
| `area` | 目标面积；不适用时为 `0` | `0` |
| `angle_deg` | 巡线或标签角度；不适用时为 `None` | `None` |

统一坐标系暂定为显示坐标 `640 x 480`，画面中心为 `(320, 240)`：

```python
dx = cx - 320
dy = cy - 240
```

因此 `dx < 0` 表示目标在画面左侧，`dy < 0` 表示目标在画面上方。这只是视觉坐标约定；云台、电机或舵机的实际正负方向必须等联调时实机确认。

## 各模块的转换规则

| 模块 | `target_type` | `confidence` 来源 | 特别规则 |
| --- | --- | --- | --- |
| 色块 | `red`、`green`、`blue` 等 | 由面积/稳定帧数折算 | 必须先按任务规则选定一个目标。 |
| 圆形 | `circle` | 检测质量或稳定帧数 | `area` 可使用 `pi*r*r` 近似。 |
| AprilTag | 例如 `tag36h11:5` | 检测成功为 100，或后续按稳定帧数折算 | `angle_deg` 使用 Tag 旋转角。 |
| 巡线 | `line` | ROI 有效数量或稳定帧数 | `dx` 使用巡线误差，`dy=0`。 |
| 在线训练检测 | 训练类别，如 `watch` | 原始检测置信度乘 100 | 多框时先按任务规则选择一个框，不能随意发送全部框。 |

## 现场标定参数（比赛时必看）

| 参数 | 含义 | 什么时候调 |
| --- | --- | --- |
| 统一坐标系尺寸 | 输出给整机的坐标参考尺寸 | 只有显示或处理尺寸整体变化时调整；改动后必须重新检查 `dx/dy`。 |
| 画面中心 | `dx/dy` 的零点 | 相机安装位置或裁剪 ROI 改变后调整。 |
| 多目标选择规则 | 同时出现多个候选时的唯一目标规则 | 按题目要求确定，例如最大面积、最高置信度、最近中心或指定类别。 |
| `valid` 稳定帧数 | 连续几帧识别到才判为有效 | 误触发时增加；响应过慢时减少。默认先由各模块保持单帧调试，实机整合时再定。 |

调试顺序：先固定相机和坐标系 -> 验证每个单项模块的 `cx/cy` -> 检查 `dx/dy` 左右上下符号 -> 定义多目标选择规则 -> 最后加入稳定帧和丢失目标行为。UART 只在这些步骤稳定后接入。

## 下一步实现
