# K230 视觉库路线图

## 工作规则

不需要先学完所有教程才能写有用代码。每个能力都按下面的短闭环完成：

1. 先运行对应的亚博源码，弄清输入、输出和参数含义。
2. 写一个可以被 CanMV IDE 直接运行的独立示例，带预览、调试叠加层和可现场调整的常量。
3. 用实物验证。若基础 API 不稳定，立即换成更好用的实现，如 CV Lite，不等到后面再返工。
4. 通过实物测试后，才提取为公用竞赛接口和 UART 数据包。

独立示例始终是第一线的排错工具；集成程序只由实测通过的示例组成。

## 近期顺序

1. 重新验证矩形底层检测，再用固定 ROI 和几何过滤改进矩形模块；当前全帧演示不视为最终解法。
2. 圆形模块保持 RGB888 CV Lite 基线，后续只按实际圆靶尺寸标定参数。
3. 在实际跑道上标定巡线，在交给控制组前固化误差正负号约定。
4. 有具体复杂靶标且传统视觉不足时，训练专用模型并用在线部署 Demo 验证；没有明确目标时，不提前训练无关模型。

## 最新状态

COCO YOLOv8 代码已移至 `vision/reference/yolo_coco_pipeline/`，只作为 KPU 参考，不属于比赛可直接使用的 Demo。

第 10 模块保留亚博 `13.training/deploy_det_video.py` 的原样部署 Demo。它读取在线训练平台导出的 `AnchorBaseDet` 模型与 JSON 配置，操作者已确认原例程可直接运行；每个新模型仍需单独完成实机复测。

在线训练部署链路已完成 `watch` 的采图、标注、训练和板端识别闭环。下一步是统一视觉结果格式，再建立仅在屏幕调试的模式切换集成工程。

## 完成定义

一个模块不是画出检测框就算完成。它必须具有：可运行示例、调参说明、实物测试结果、明确的无效/丢失输出，以及提供给控制组的输出格式说明。

## 文档约定

从现在开始，每个新增视觉模块的 Markdown 都必须包含“现场标定参数（比赛时必看）”一节：列出参数、含义、何时调整和固定调试顺序。若模块没有可调算法参数，也要明确写出“不调参”及原因，不得留空。
